# Design-sync notes

- Repo is a private Next.js app (`tecta`), not a published component library — no `main`/`module`/`exports` in package.json, no `dist/`. Using synth-entry mode: converter scans `components/ui/*.tsx` directly (last resort per skill docs — weaker `.d.ts` extraction than a real build).
- Scope: `components/ui` only (~23 shadcn/radix-based primitives). The rest of `components/` (hero, navbar, wallet-provider, three.js/canvas effects, etc.) are page-specific compositions, not reusable primitives — excluded from this sync.
- Styling: Tailwind v4 + shadcn "radix-nova" style, CSS variables (`app/globals.css`), no prefix.
- Path alias `@/*` → repo root (tsconfig.json), needed for esbuild resolution in synth-entry mode.

- Excluded `CanvasRevealEffect`, `FaultyTerminal`, `CardSpotlight`, `CardSpotlightDemo` via `componentSrcMap: null` + a forked `.design-sync/overrides/source-kit.mjs` (`ENTRY_EXCLUDE` regex — componentSrcMap alone doesn't stop a file from entering the synth entry's `export *` barrel). Root cause: `canvas-reveal-effect.tsx` imports `@react-three/fiber`, which imports `scheduler` directly; since the whole DS bundles into one shared IIFE, that top-level import evaluates on load and trips the bundler's `[SCHEDULER_MISSING]` guard for the *entire* bundle, breaking every component's render, not just these two. `card-spotlight.tsx`/`card-spotlight-demo.tsx` transitively re-import `CanvasRevealEffect` (it's their hover-reveal effect), so they had to be excluded too. All four are WebGL/canvas-driven or thin wrappers around one — not meaningful as static preview cards anyway (need live mouse interaction), so excluding them costs nothing for a design-system sync.

- `app/globals.css` is raw Tailwind v4 input (`@import "tailwindcss"`, `@import "shadcn/tailwind.css"`, etc.) — a browser/converter can't resolve those as literal file imports, only the Tailwind build step can. `cfg.cssEntry` points instead at `.design-sync/.cache/compiled-tailwind.css`, generated by `node .design-sync/.cache/compile-tw.mjs` (postcss + `@tailwindcss/postcss` API, JIT-scans the whole repo for used classes). **Re-run that script before every build/rebuild** — it's gitignored machine state, not committed, and goes stale if globals.css or any component's class usage changes.

## Re-sync risks
- Synth-entry mode means `.d.ts` prop contracts are structural guesses, not a real build's output — re-verify after any shadcn component upgrade.
- No storybook/stories in this repo — previews are 100% authored by design-sync, no reference render to diff against.
